### 里程碑 1：环境极速就绪与 BIRD 基准打通
**核心目标**：抛弃繁重的工程搭建，在 VS Code 中配置好 Python 环境（如 Anaconda 虚拟环境），跑通大模型调用与 BIRD 数据的读取。

1. **基础脚手架（Vibe Coding 生成）**：
   * 编写 `QwenClient`，对接云端 Qwen API，封装好带有重试机制和 `temperature` 控制的对话接口。
   * 编写脚本加载 BIRD-DEV 数据集（包含 `dev.json` 和对应的 SQLite 数据库文件）。
2. **安全拦截器基线**：
   * 利用 `sqlglot` 库编写 `SafetyInterceptor`，实现对 AST 的解析，精准拦截 `DROP`、`DELETE` 等危险关键字，确保哪怕是在测试阶段也具备防御意识。

### 里程碑 2：EvoSQL 核心引擎复现与测试
**核心目标**：实现多级迭代上下文精炼（MLICR），并在 BIRD-DEV 上验证执行准确率（EX）。既然这套底层算法逻辑是你们亲手打造并投递的，代码层面的组装只需要让 AI 按照论文逻辑精准搬砖即可。

1. **实现 SCR 与 ECA 策略**：
   * **SCR 模块**：提取候选 SQL 的 Schema 取并集，并清空结构提示。
   * **ECA 模块**：连接 BIRD 的 SQLite 进行沙箱执行，对结果聚类，提取 Top-k 簇作为高置信度锚点。
2. **主循环与自适应调度**：
   * 将上述策略封装进最多 3 轮的迭代循环中。
3. **闭环跑测**：
   * 在 BIRD-DEV 上进行小批量测试，确保执行逻辑不崩溃，并记录初步的 EX 指标，验证代码行为是否与预期一致。

### 里程碑 3：校园私有域数据合成与迁移
**核心目标**：完成从公开数据集到北航真实业务场景的“领域迁移”。

1. **抽取校园 Schema**：
   * 提炼几张核心的校园业务表结构（例如：包含学院、专业信息的教务表，科研项目表，学生成绩表等），做好脱敏处理。
2. **大模型逆向合成数据**：
   * 编写 Prompt，将这些 Schema 喂给能力最强的模型（如 GPT-4o 或 DeepSeek-R1），让其作为“高校数据分析专家”，逆向生成 50-100 对高质量的 `(自然语言问题, 标准 SQL)` 数据对。要求涵盖多表关联、嵌套查询等复杂逻辑。
3. **引擎迁移测试**：
   * 将 EvoSQL 引擎的数据源从 BIRD 切换至你模拟的校园 SQLite 数据库，测试算法在自建数据集上的表现与 Schema 链接去噪能力。

### 里程碑 4：全栈应用拼装
**核心目标**：为算法套上壳子，做成一个真正能演示的交互式产品。

1. **后端服务编排**：
   * 用 Flask 快速起一个 Web 服务，暴露问数 API。接入任务队列处理大模型的高时延推理。
   * 增加 Data-to-Text 模块：将底层 SQL 查出的结果，再次扔给 Qwen API 生成一段自然语言解释。
2. **前端交互层开发**：
   * 使用 Vue.js 搭建极简的类似搜索引擎的问数界面。
   * 集成 ECharts 等组件，根据后端返回的数据特征（时间序列、占比等），自动渲染折线图或饼图。
